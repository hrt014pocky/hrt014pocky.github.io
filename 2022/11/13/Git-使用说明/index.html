Git 使用说明

发表于 2022-11-13 更新于 2023-10-23 分类于 Git

Git安装

安装完成后，查看Git版本，检查是否正确安装。

Git的基础设置

Git的设定被存放在用户本地目录的.gitconfig档案里。虽然可以直接编辑配置文件，但在这个教程里我们使用config命令。

config配置有3个层级：

  • system（系统级别）
  • global（用户级别）
  • local（仓库级别）

覆盖优先级先从system->global->local。

创建一个Git仓库

首先在任意一个地方创建代码仓库目录gitTest。然后使用init命令把该gitTest目录移动到本地Git数据库。

克隆一个仓库

如果需要克隆远程仓库，可以使用git clone。

文件状态

Git中每个文件都有三种状态：committed、staged、modified。

  • commit <-- stage <-- modify
  • commit --> — --modify
    我们获取的Git仓库中的所有文件都是committed状态，如果你在本地修改了文件a，a的状态就变成modified的；如果使用git add a，a的状态变成staged；如果使用git commit，a的状态就变成commited。
    还有一种未跟踪状态（unversioned/untracked），通过使用git add可以把未跟踪状态变更为staged；通过git rm可以将staged或者committed状态变为未跟踪状态。
    git rm会把文件从当前目录删除（不会保存删除的文件）。如果需要从Git仓库中删除，但保留在当前工作目录中，亦即从跟踪清单中删除，可以使用git rm --cached readme.md

查看当前修改

git diff可以查看当前目录的所有修改。可以使用git diff --staged或git diff --cached查看staged与上次提交快照之间的区别。

添加远程仓库

可以给远程数据库取一个别名。这样，下次推送的时候就不需要输入长串的远程数据库地址了。在这里，我们的远程数据库命名为“origin”。

从远程仓库抓取数据

也可以使用git pull

  • git pull 是从远程获取最新版本并merge到本地仓库
