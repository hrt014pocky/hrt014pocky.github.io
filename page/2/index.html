Git安装

安装完成后，查看Git版本，检查是否正确安装。

Git的基础设置

Git的设定被存放在用户本地目录的.gitconfig档案里。虽然可以直接编辑配置文件，但在这个教程里我们使用config命令。

config配置有3个层级：

  • system（系统级别）
  • global（用户级别）
  • local（仓库级别）

覆盖优先级先从system->global->local。

创建一个Git仓库

首先在任意一个地方创建代码仓库目录gitTest。然后使用init命令把该gitTest目录移动到本地Git数据库。

文件状态

Git中每个文件都有三种状态：committed、staged、modified。

  • commit <-- stage <-- modify
  • commit --> — --modify
    我们获取的Git仓库中的所有文件都是committed状态，如果你在本地修改了文件a，a的状态就变成modified的；如果使用git add a，a的状态变成staged；如果使用git commit，a的状态就变成commited。
    还有一种未跟踪状态（unversioned/untracked），通过使用git add可以把未跟踪状态变更为staged；通过git rm可以将staged或者committed状态变为未跟踪状态。
    git rm会把文件从当前目录删除（不会保存删除的文件）。如果需要从Git仓库中删除，但保留在当前工作目录中，亦即从跟踪清单中删除，可以使用git rm --cached readme.md

查看当前修改

git diff可以查看当前目录的所有修改。可以使用git diff --staged或git diff --cached查看staged与上次提交快照之间的区别。

Hexo框架搭建

Hexo 是一个基于Nodejs的博客框架，快速、简洁且高效，可以方便的生成静态网页托管在GitHub或者Gitee上面。在安装Hexo之前首先需要确保自己的电脑已经正确安装了Git和Nodejs。

Git安装

Git安装下载地址：https://git-scm.com/downloads

  • windows:Git官网下载并安装。
  • Mac：使用 MacPorts 或者下载安装程序。
  • Linux (Ubuntu, Debian)：sudo apt install git-core
  • Linux (Fedora, Red Hat, CentOS)：sudo yum install git-core

Nodejs安装

官方网站下载安装包进行安装 https://nodejs.org/en/download/

使用Github建立博客网站

以上我们在以及建立了一个博客框架，但是只能在本地访问。想要在互联网上查看我们写好的文章，还需要部署到网站上。在里我使用的是Github Pages，它提供了静态网页托管服务，无须自己购买云服务进行搭建，并且是是完全免费的。

Github账号注册与代码仓库创建

  1. 进入Github官网，点击Sign up进行注册。
  2. 进入代码仓库 Your repositories ,点击New创建一个新的仓库。
  3. 注意代码仓库的名称为 “[你的Github用户名].github.io”，点击Create repositories新建代码仓库。
    4.进入创建的代码仓库，点击Create new file新建一个index.html的文件。输入内容
  1. 进入创建的代码仓库，选择Settings,打开Pages选项，点击GitHub Pages
  2. 浏览器打开一个网页，输入网站：https://[你的Github用户名].github.io/ ，如果显示test文字说明创建Github Pages个人主页成功，如果显示找不到网页，说明搭建失败。
